Step-by-Step Walkthrough

1

Prepare Your Context

Before touching any builder tool, you need a compass. Define your product's core identity using the Context Engineering templates from Module 5.

üéØ Mini-PRD Prompt builder

Can't think of a PRD? Ask an AI to generate one for you using this prompt chaining technique:

I am building a marketing website for [My Product/Service]. 
Act as an expert Product Manager. Please generate a concise 1-page PRD including:
1. Target Audience
2. Core Value Proposition
3. Key Features (Top 3)
4. Recommended Color Palette (Hex codes) & Typography
2

Chain Your Initial Prompt

Now, take the output from your PRD and merge it into a highly specific architectural prompt for your builder (Lovable, Bolt, etc.).

üîó Master Prompt Template
Build a modern, responsive landing page for [Product].
                                    
DESIGN SYSTEM:
- Primary Color: [Hex]
- Font: Inter (or similar modern sans-serif)
- Style: Dark mode, glassmorphic elements, glowing borders.

REQUIRED SECTIONS:
1. Hero: Headline, subheadline, 2 CTA buttons.
2. Features: 3-column grid with hover animations.
3. Pricing: 2 tiers, highlight the "Pro" tier.
4. Footer: Links and copyright.
3

Generate & Iterate

Paste your Master Prompt into your chosen builder tool. Don't worry about perfection on the first try. The magic of vibe coding is in the iteration.

üí° How to Fix Errors

If a layout looks weird or the colors are off, be highly specific in your follow-up prompt: "Make the feature cards equal height. Change the hero background gradient from blue to purple (#8B5CF6 ‚Üí #6D28D9)."

4

Add Polish & Deploy

Request animations, hover states, and responsive adjustments. Test on mobile views using browser DevTools. Once it looks professional, hit Deploy.

- Add a subtle staggering fade-in animation to the feature cards on scroll.
- Ensure the hero typography scales down appropriately for mobile screens.
- Make the primary CTA button pulsate softly to draw attention.
