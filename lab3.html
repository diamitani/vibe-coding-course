Project Brief

üè™ Directory / Marketplace Platform

Build a multi-listing platform for a niche of your choice. This could be a local restaurant directory, a freelancer marketplace, a SaaS tools directory, an AI tools aggregator, or anything with multiple listings that users can browse, search, and filter.

Requirements

  • Homepage ‚Äî Hero with search bar, featured/popular listings, category navigation
  • Listing Grid ‚Äî Cards showing key info (name, category, rating, image/icon, short description)
  • Search ‚Äî Real-time text search across listing names and descriptions
  • Filters ‚Äî Filter by category, rating, price range, or other relevant criteria
  • Detail Page ‚Äî Full listing profile: description, features, contact/CTA, reviews or ratings
  • Submit Listing ‚Äî A form for new listings to be added
  • Responsive ‚Äî Works on mobile and desktop

Recommended Tools

Lovable (best for full-stack + database), Replit, or the advanced combo of Claude Code / Antigravity + a framework.

üöÄ The Vibe Prompt

Standardize your logic in the Prompt Lab first:

"Build a comprehensive directory for SaaS tools. Required: Real-time search that filters as you type, category pills for quick navigation, a detailed profile page for each tool, and a submission form with validation. The design should be emerald-themed, professional, and dense with information."

Step-by-Step Walkthrough

1

Design the Directory Schema

A marketplace lives and dies by its data. Define a robust schema for your listings before anything else, and generate 15-20 data seeds.

üß† Directory Schema Prompts
I am building a comprehensive directory for [My Niche]. 
Please generate a JSON array with 15 mock listings.
Each listing must contain:
1. id (string)
2. name (string)
3. category (string, e.g., "SaaS", "Agency")
4. rating (number, 1-5)
5. tags (array of 3 strings)
6. description (2 sentences)
7. websiteUrl (string)
2

Architect the UI & Search

Prompt your builder to set up the core layout: a search bar, filtering logic, and the listing grid. This is the hardest prompt to get right.

üèóÔ∏è Core Marketplace Prompt
Build a full-stack directory application using React.
1. Use the 15 mock JSON listings I provide below as initial data.
2. Build a Hero section containing a large, centered real-time search bar.
3. Search Logic: Filtering must happen instantly as the user types, checking both 'name' and 'description' fields.
4. Filter Logic: Add a sidebar with checkboxes for 'category' and a slider for 'minimum rating'.
5. UI Layout: Use a dense 3-column CSS grid for the listing cards. Theme it with an emerald green accent.
3

The Submission Form

A directory isn't useful if it can't grow. Ask your AI to create a standalone form that allows users to submit new listings.

üí° How to handle new data

If you're lacking a true backend (like Supabase), prompt the AI specifically: "When a user submits the form, validate that all fields are filled, then unshift the new object into the 'listings' array stored in window.localStorage."

4

Polish, Empty States & Detail Pages

The mark of a professional marketplace is how it handles edge cases. What happens when a search returns zero results? What happens when you click a card?

‚ú® Final Polish Prompts
- Add a beautifully designed empty state with an SVG illustration when a search query returns 0 results.
- Implement pagination or an "infinite scroll" intersection observer so only 9 cards load at a time.
- Clicking a card should open a modal (or dedicated page) showing the full details and tags of the listing.
Finish Walkthrough üéâ
Example Refined Prompt (After Chaining)
"Build a directory of AI tools. Data model: { name, category (Chatbot/Image/Code/Video/Audio), description, pricing (Free/Freemium/Paid), url, rating (1-5), tags[] }. Homepage: search bar at top, category pills below, featured grid of top-rated tools. Listing grid: cards with tool name, category pill, rating stars, pricing badge, one-line description. Clicking a card opens a detail page with full description, feature list, pricing details, link to tool, and similar tools section. Filter sidebar: category checkboxes, pricing radio buttons, minimum rating slider. Submit form: all fields + tag input. Persist in localStorage. Dark theme, Inter font, #8B5CF6 accent, glassmorphism cards, hover animations."

Completion Checklist

  • Homepage with search bar and category navigation
  • Listing grid with at least 10 listings
  • Real-time search functionality
  • At least 2 filter criteria (category, price, rating, etc.)
  • Detail page for individual listings
  • Submit listing form with validation
  • Responsive design (mobile + desktop)
  • Professional card design with hover effects
  • Empty states and edge case handling
  • Site deployed and accessible via public URL
