Lab 2

Build an E-Commerce Website

Level up your vibe coding skills by building a complete online store. This lab introduces product data, state management, and interactive elements like a shopping cart and checkout flow.

  • Project Brief
  • Walkthrough
  • Checklist

Project Brief

üõí Online Store

Build a functional e-commerce store selling a product category of your choice (sneakers, coffee, tech gadgets, etc.). The store should feel real enough to impress a client.

Requirements

  • Homepage ‚Äî Featured products, hero banner, category sections
  • Product Catalog ‚Äî Grid of products with images, names, prices
  • Product Detail Page ‚Äî Full product view with description, images, "Add to Cart" button
  • Shopping Cart ‚Äî Add/remove items, quantity adjustment, order total
  • Checkout Flow ‚Äî Form with shipping and payment fields (no real payments needed)
  • Search & Filter ‚Äî Filter by category or sort by price
  • Responsive ‚Äî Fully responsive from mobile to desktop

Recommended Tools

Lovable (best for full-stack), Bolt.new, or Replit. For more advanced: Claude Code or Antigravity.

Step-by-Step Walkthrough

  1. Choose Your Product Category

    Pick something you're interested in. Create 8‚Äì12 products with names, descriptions, prices, and categories. You can use AI to generate the product data.

  2. Create Your Context Package

    Write a PRD that defines: product schema (name, price, description, category, image), page structure, cart behavior, and checkout fields. Include your design system.

  3. Chain Your Master Prompt

    Use prompt chaining to refine a detailed prompt. Include product data, layout requirements, color scheme, and interactive behaviors (cart slide-out, quantity buttons, etc.).

  4. Generate the Store

    Use your chosen tool. Start with the homepage and product grid, then iterate on the detail page, cart, and checkout.

  5. Add Product Data

    If the tool supports databases (Lovable+Supabase, Replit), use a real data store. Otherwise, use JSON data or localStorage.

  6. Polish the Cart & Checkout

    The cart should feel smooth: slide-in/out animation, real-time total calculation, quantity +/- buttons. Checkout should validate fields and show a confirmation.

  7. Test & Deploy

    Test the full flow: browse ‚Üí add to cart ‚Üí adjust quantity ‚Üí checkout. Test on mobile. Deploy.

Example Refined Prompt (After Chaining)
"Build an e-commerce store for premium headphones. Products: [list 8 products with name, price, category, and one-line description]. Homepage: hero with featured product, 'Shop by Category' section (Over-Ear, In-Ear, Wireless), product grid with hover effects. Product page: large image, title, price, description, reviews count, 'Add to Cart' with quantity selector. Cart: slide-in from right, list items with quantity +/-, subtotal, 'Checkout' button. Checkout: form with name, email, address, card fields. Success state: confetti animation + order number. Dark theme, #0A0A0F background, violet accent #8B5CF6. Inter font. All data in localStorage."

Completion Checklist

  • Homepage with hero and featured products
  • Product grid with at least 6 products
  • Product detail page with description and "Add to Cart"
  • Working shopping cart (add, remove, quantity)
  • Checkout form with basic validation
  • Search or filter functionality
  • Responsive design (mobile + desktop)
  • Consistent design system throughout
  • Hover effects and transitions
  • Site deployed and accessible via public URL
